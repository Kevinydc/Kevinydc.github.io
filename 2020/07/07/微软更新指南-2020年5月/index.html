    微软更新指南 - 2020年5月[In progress]

    Posted on 2020-07-07 In 微软更新指南

    2020年5月更新

    文章翻译自May 2020 release

    对于微软来说，5月又是一个忙碌的月份。F&O有许多值得注意的发布，电子商务部分也有许多重要的更新，Power Platform增加了好几个易用性方面的改善。请特别注意以下内容的前几条，他们对正在进行的项目的健康状况至关重要。

    May was another busy month for Microsoft. F&O had some notable releases, Commerce had important numerous updates and Power Platform added several usability improvements. Pay special attention to the first few entries below as they are critical to ongoing project health.

    Finance and Operations

    基础更新

    随着微软引导客户使用更自助式的模式，环境管理的几个根本性变化也在进行中，技术架构师(TA)应该密切关注这一块儿。

    Several fundamental changes to environment management are underway as Microsoft guides customers to a more self-service model. Technical Architects should pay close attention.

    删除了对2 - 5层的SAT环境的远程桌面访问

    所有新实例都不能访问第2层VM。现有实例将在9月份失去访问权。LCS是调查事件日志、SQL问题和性能问题的主要来源。

    Access to Tier 2+ VM’s is going away for all new instances. Existing instances will lose access in September. LCS is the primary source to investigate event logs, SQL issues, and performance problems.

    支持为自助服务环境应用包的APIs

    向自助服务环境的转变将取消使用Azure DevOps任务进行夜间构建过程和CI/CD管道的能力。对于1-5层沙箱VM的包应用程序将只存在于LCS中。

    The move to self-service environments will remove the ability to use Azure DevOps tasks for nightly build processes and CI/CD pipelines. Package application will live exclusively in LCS for Tier 1-5 sandbox VM’s.

    通过学习RESTful APIs for database export为这个变化做好准备。尽管变化很大，但是这一变化也增加了自动将数据库从PROD回复到SAT的可能性。

    Get ready for the change by learning about RESTful APIs for database export. Although the change is significant it also adds a quality of life feature via the ability to automate database restore from PROD to SAT.

    LCS问题搜索改进

    希望你们不需要经常搜索基础的产品问题😉，但是你可能已经注意到，微软改进了流程。LCS现在提供附加的信息：

    Hopefully, you all aren’t having to research base product issues too often 😉, but you may have noticed that MSFT improved the process. LCS now provides additional information for:

    • 影响版本的发布
    • 发布类型指示每个发布版本如何可用，可以通过质量更新或服务更新
    • 每个计划发布的状态
    • 每个计划发布的可用性
      • Release of the impacted versions
      • Release type indicating how each release will be made available, either via quality update or service update
      • Status of each planned release
      • Availability of each planned release

    供应链管理

    生产主计划支持采购贸易协议

    新的主计划优化服务即将到来！同时还带来了很多新的功能，新的服务在主计划运行的时候能够考虑采购贸易协议在最短提前期和最低价格的前提下选择最合适的供应商。文档里有很好的举例和解释。

    The new planning optimization service is coming! And, it’s going to have some great new features. The new service can consider purchase trade agreements to select the best vendor based on the shortest lead time or lowest price.

    关于新的主计划功能和服务相关内容，请重读2020年4月份更新指南

    D365供应链管理客户门户

    公司可以通过客户门户创建一个用于B2B场景的网站，公开一些像订单状态这种通常会被请求的信息。这个功能只是一个模板，不是一个完整成熟的解决方案。项目团队应该对此功能进行研究，因为此功能解决了客户的常见需求。因为你是在为一个基于页面浏览量收费的Power门户付费，所以肯定要考虑许可问题。

    Customer portals allow companies to create websites for B2B scenarios and expose commonly requested information like order status. The feature is a template, not a fully baked solution. Project teams should investigate as the feature solves common client requests. There are definitely licensing implications to consider because you are paying for a Power Portal which is charged based on page views. Check out the below video!

    仓库管理库存量记录清理批处理最大执行时间参数

    任何项目在实施高级仓库管理模块的时候，都需要周期性的运行库存量记录清理批处理作业。它用于清除一些重要表中的“杂物”，这些表经常被用于计算。新功能通过一个简单的超时参数，在作业持续运行太长时间时保存已处理的记录，这样能够更好的选择运行时间，在系统负载尽可能轻的时候运行。

    Any project team implementing advanced WMS should be running the on-hand cleanup job periodically. It clears “clutter” out of important tables that are constantly hit for calculations. The new feature is a simple time-out parameter that saves the processed records in the event that the job consistently runs too long.

    报表改进

    我们注意到，在过去的几个月，微软对一些非常大的报表进行了加工。某些报表提示用户输入一个具体的名字并保存在系统中，这样从根本上降低了系统的压力因为放弃了对相同数据的重新生成。2020年1月更新指南已经提到过库龄报表Inventory aging report，这次是现有库存量报表On-hand inventory report。我们都知道它得运行多久😎

    We’ve noticed a trend over the last few months of Microsoft retooling large reports. Certain reports prompt users for a named value and are saved in the system which ultimately reduces system strain because it forgoes a regeneration of the same data. The January Compass mentioned the inventory aging report. May brought the same feature to the On-hand inventory report. We all know how long that can take to run 😎.

    入站负载管理增强

    跳转到上面的链接，查看所有入站管理增加的新功能。值得注意的一点是，增加了在相同负载下过账多个收货单的功能！

    Jump to the above link to see all the new features for inbound loads. A notable one adds the ability to post multiple product receipts against the same load!

    在消息栏中嵌入操作

    酷酷的AX 2012功能回归了😎，允许开发者在状态栏的警告或错误信息中嵌入动作按钮。

    The return of a cool AX 2012 feature that allows developers to embed action buttons in warning or error messages on the status bar. For example, an item setting may cause a process to fail. You can now embed a link to the product master and update that setting. Might be a neat way to direct users during process flow variations that require action!

    贸易

    零售贸易部分有一堆适用于我们项目的新功能，仔细看看并且和你的团队讨论。

    Commerce has a bunch of new features that are applicable to our project teams. Take a close look and discuss with your team!

    POS端修改送货方式的操作优化

    POS端一个更加直接的方法更新单独订单行的送货方式。项目团队应该知悉这一点改进，因为这一点在我们的试试过程中被长期诟病。

    A more direct way to update individual order line mode of delivery in the POS. Project teams should be aware of this improvement. It was a longstanding gripe on our implementations!

    自动费用的改进支持基于渠道上的费用计算

    自动费用功能的一个扩展，允许费用按照渠道层级。这一功能本质上允许商店有他们自己的自动费用规则。

    An extension of the auto-charge functionality to allow charges by Channel hierarchy. The feature essentially allows stores to have their own auto-charge rules.

    Azure AD支持POS端登录验证

    正如标题所说的，用户可以用他们的Azure AD的账号登录POS。这也是微软带来的跨平台便利性很好的例子。

    Exactly as the title implies, users can now use their Azure Active Directory credentials to login at the POS. Another good example of cross platform convenience by MSFT.

    电子商务和呼叫中心支持外部的礼品卡

    对Givex和SVS进行开箱即用的集成，这将为团队节省大量时间!我知道Givex在过去造成了一些痛苦。

    Out-of-the-box integration to Givex and SVS that should save teams a good chunk of time! I know Givex has caused some pain in the past.

    POS端可通过邮件再次发送收据

    在POS端增加了可以对已关闭交易再次发送收据的功能，同时可以指定新的邮件地址。

    Adds the ability to resend a receipt from a closed transaction on the POS and specify a new email address.

    电商里的库存缓冲 - 公开预览

    你可能已经为每一个电子商务集成设计了库存缓冲，所以很高兴看到微软带来了这个功能。实时网站库存查询是不现实的。创建一个库存缓冲区可以提醒用户“库存不足，请尽快订货”，而不会不准确地报出数据，从而影响系统性能。

    You’ve likely designed inventory buffers for every eCommerce integration so it’s great to see MSFT bringing it their offering. Real-time website inventory calls isn’t realistic. Creating an inventory buffer allows the website to warn users “Stock is low, order soon.” without inaccurately quoting the true number and impacting system performance.

    Power Platform

    Power BI Desktop

    上个月发布了一堆的Power BI桌面版功能。下面这个视频提供了一个非常好的例子关于如何搭建一个类似APP体验的Power BI报表。观看视频或者留意下面的亮点总结！

    A bunch of new Power BI Desktop features dropped last month. The below video provides a great example of how you can build rich app-like reporting experiencing in Power BI. Watch the video and/or check out the highlights below!

    钻取按钮

    可以通过按钮，打开需要钻取的报表。

    部署途径 - 开发，测试，和部署

    这个高级特性最终增加了一种管理开发和部署生命周期的方法。

    The Premium-only feature finally adds a way to manage the development and deployment lifecycle.

    共享和认证的数据集

    Power BI的一个危险是，人们可以发布相互冲突的数据集供使用。认证的数据集允许一个主要的受信的组创建和分享一个确实是真实来源的数据集。

    A danger of Power BI is that people can publish conflicting datasets for consumption. Certified datasets allow a central, trusted group to create and share a dataset that can serve as the source of truth. A good video overviews it briefly.

    Power Apps

    画布应用和模型驱动应用运行在一个移动应用上

    画布应用和模型驱动应用本质上不同的，它们的使用方式也不同。新的功能允许这两种应用通过同一个移动应用使用。

    Canvas apps and model-driven apps are inherently different and were previously consumed differently. The new feature allows model-driven apps to be consumed through the same mobile application as canvas apps.

    在对话窗中创建和编辑数据

    用户不再需要导航到其他地方去添加或编辑相关记录。制作者可以直接在对话框中从另一个主窗体打开一个主窗体。这在处理相关记录时减少了单击和不必要的导航，从而极大地提高了整体生产力。这个特性将与”navigateTo” api一起提供，可以从命令栏、事件或插件中使用，在对话体验中打开主表单记录。

    Users no longer have to navigate away to add or edit a related record. Makers will be able to open a main form in a dialog directly from another main form. This greatly improves overall productivity by reducing clicks and unnecessary navigation when working with related records. This feature will be available with the ‘navigateTo’ api and can be used from a command bar, event, or plugin to open a main form record in a dialog experience.

    新的环境配置完全过渡到Power Platform管理中心

    老的D365管理中心将被弃用，这有利于使用新的Power Platform管理中心在一个地方统一管理。

    The old D365 admin center will be deprecated in favor of the Power Platform admin center to unify environment management to a single place. Read more about the new management procedures here.

    人工智能

    微软每个月都在更深入地将人工智能融入其产品和服务中。

    欺诈防护

    值得注意的是，D365 Commerce开始提供在结帐时与欺诈保护的开箱即用集成。

    It’s important to note that D365 Commerce is beginning to offer out-of-the-box integration with Fraud Protection at checkout.
